# 2025-06-30

Countable 챌린지 아이템의 결과 표시 개발을 했다.<br>
정확히는 Over 타입만 끝났다.<br>
Under 타입은 어떻게 표시해야 사용자에게 직관적일지 고민 중인데, 아래는 Perplexity 의 답변이다.<br>

```
1. 역방향 도넛 차트(Progress Ring, Inverted Donut Chart)

목표치(최대 허용값)를 100%로 잡고, 사용자가 실제로 사용/달성한 값을 채워나갑니다.

달성률이 높을수록(=목표에 가까워질수록) 경고 색상(예: 빨강)으로 변환하여, 목표치에 근접하면 위험 신호를 직관적으로 전달할 수 있습니다.

도넛의 남은 부분이 많을수록 긍정적인 컬러(예: 500 컬러), 채워진 부분이 많을수록 경고 컬러(예: 200→300→500)로 표현합니다.

2. 게이지 바(Gauge Bar) 또는 온도계 스타일

수평 또는 수직 바로 현재 수치를 표시하고, 목표치에 가까워질수록 색상이 점점 경고색(예: 노랑→주황→빨강)으로 변합니다.

목표치 이하일 때는 바가 짧고, 초과하면 바가 가득 차면서 애니메이션이나 진동 등으로 경고를 줍니다.

3. 남은 여유(잔여치) 강조

“오늘 남은 담배: 3개”처럼 남은 여유량을 숫자와 함께 시각적으로 강조합니다.

여유가 넉넉할 때는 안정적인 컬러(예: 500), 여유가 줄어들수록 색상이 변하거나 아이콘이 바뀝니다.

4. 아이콘 및 이모지 활용

목표치 이하이면 웃는 얼굴, 파란색 체크 등 긍정적 아이콘 표시

목표치에 가까워지면 경고 아이콘(느낌표, 노란색 삼각형 등), 초과 시 실패 아이콘(빨간 X, 화난 얼굴 등)

5. 원형 타이머/카운트다운

목표치에서 현재값을 뺀 잔여량을 원형 타이머처럼 시각화

시간이 남았을 때와 비슷하게, 남은 여유가 줄어들수록 색이 변합니다.

6. 텍스트 + 시각화 조합

“목표치의 60%만 사용했어요!” 등 직관적 텍스트와 함께 도넛/게이지/아이콘을 조합하여 정보 전달
```

위를 참고하도록!

# 2025-05-08

이제 Solid JS 와 좀 많이 친해진 것 같다.<br>
이 친구는 fine grained reactivity 를 위해서 값이 변경된 곳을 추적하기 위해,<br>
데이터를 `Accessor<T>` 형태로 구성해서 실행해야한다.<br><br>
깨달았으니, 고쳐야겠지?<br>
props 를 전체적으로 돌아다니면서 `T` 데이터를 `Accessor<T>` 로 변경하고<br>
컴포넌트 타입 규격을

```tsx
type Props = {...}

const WinYourself = (props: Props) => {
  return ...
}
```

에서

```tsx
type Props = {...}

const WinYourself: Component<Props> = (props) => {
  return ...
}
```

로 변경해볼 예정이다.
